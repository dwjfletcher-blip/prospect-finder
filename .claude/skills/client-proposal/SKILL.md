# Skill: Client Proposal

Generate a professional project proposal ready to send to a prospective client.

## When to Use

When David has a specific client or lead in mind and needs a written proposal to move the conversation forward. Run after project-brief if possible — the brief feeds directly into this.

## How to Run This Skill

1. Ask for the inputs listed below. Ask all at once.
2. If a project brief already exists, use it — skip any questions already answered.
3. Generate the proposal using the output format below.
4. Always end with a pricing note (see guidance below).

## Inputs Needed

- Client name and business name
- What they need (or pull from existing project brief)
- Any specific pain points they mentioned
- Desired timeline or deadline (theirs or David's estimate)
- Proposed price — if David doesn't know, run pricing guidance first (see below)
- Revision rounds included
- Preferred contact / next step (reply to email, schedule a call, etc.)

## Pricing Guidance (run this if David hasn't set a price)

Ask:
- Is this a website, web app, or something else?
- Roughly how many pages or screens?
- Any custom features (forms, booking, ecommerce, etc.)?
- Does the client need hosting/domain setup included?

Then recommend a price using this logic:
- Simple 1-5 page website, no custom features: $500–$1,200
- 5-10 page website with contact form or basic features: $1,200–$2,500
- Custom web app or complex functionality: $2,500–$6,000+
- Add $100–$200 if hosting/domain setup is included
- David is early-stage — price confidently but don't undercut so hard it signals low quality. A low price with a clear scope is better than a vague price.

## Output Format

---

**PROPOSAL**
**Prepared for:** [Client Name], [Business Name]
**Prepared by:** David Fletcher
**Date:** [Date]

---

**Overview**

[2-3 sentences. What David understands about their situation and what this project will do for them. Written from their perspective, not his.]

---

**Scope of Work**

- [Deliverable]
- [Deliverable]
- [Deliverable]

---

**Not Included**

- [Exclusion — be specific to avoid scope creep]
- [Exclusion]

---

**Timeline**

[Start date or "Upon agreement"] through [estimated completion]. Assumes client provides [any assets/content needed] within [X] days of project start.

---

**Investment**

**Total:** $[Amount]

Includes [X] round(s) of revisions. Additional revisions billed at $[hourly rate or flat fee] per round.

A 50% deposit is required to begin. Remaining balance due upon completion before final files/access are delivered.

---

**Next Steps**

[Clear single action — e.g., "Reply to confirm and I'll send over a simple agreement and invoice for the deposit." or "Let's schedule a 20-minute call to go over any questions."]

---

**Notes for David (strip before sending)**
- [Any flags — scope risks, unclear client requirements, pricing confidence notes]

---

## Hard Rules

- Never use weasel words ("hopefully", "should be able to", "try to"). Be direct and confident.
- Always include what's NOT included. Scope creep kills early-stage projects.
- Always require a deposit. No exceptions — it filters out non-serious clients.
- Keep the overview focused on the client's outcome, not David's process.
- Tone: professional, concise, confident. No filler. No emojis.
